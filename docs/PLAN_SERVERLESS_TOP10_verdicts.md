## securityVerdict

PASS on the gate; honest residual risks disclosed. ONBOARDING IS GATED, NOT A BACKDOOR: verify-before-connect is the only admission path — the scanner runs crypto.verify(pubkey, preimage, sig) over the domain-tagged pre-image BEFORE any RTCDataChannel opens; invalid/missing signature = rejected, no channel, no admission (verified intent in signaling.js:32–41 and designed onboard_verify.verify_onboarding). Anti-replay: single-use nonce + injected-integer freshness window; domain-tag separation ("knitweb-p2p-identity:v1" vs the onboarding tag, signaling.js:63) prevents an onboarding sig being replayed as an identity proof. KEY CUSTODY: identity = AccountNode.from_seed(sha256("knitweb:account:seed:"+seed)); seed in IndexedDB; no private key leaves the worker; ALL entropy from WebCrypto getRandomValues — verified never Math.random in signaling.js/store_idb.js (a predictable nonce would break the signed-QR gate). RESIDUAL HONEST RISKS: (1) browser storage is evictable — StorageManager.persist() is best-effort; mitigated by signed-state/seed export + peer re-sync (restore balance, never re-mint). (2) SYBIL is PROBABILISTIC not absolute — PoW only raises cost, scheme-0 nullifier is the trusted-RP construction (verifier sees secret at admission; trustless needs EC-VRF scheme-1), web-of-trust is collusion-gameable; tune so marginal faucet value < PoW+trust cost. (3) The onboard_verify.py / onboard.js / transport_webrtc.js gate files are DESIGNED but NOT YET WRITTEN — the verdict covers the design + the 4 existing files; the gate is unproven until those land and the L6 conformance corpus passes.
